##### 3.3 Requirements refinement:
‚ñ™ During this stage, the requirements are refined using information about the reusable components and applications that have been discovered. 
‚ñ™ The requirements are modified to reflect the available components, and the system specification is re-defined. Where modifications are impossible, the component analysis activity may be re entered to search for alternative solutions.

‚ñ™ If an off-the-shelf application system that meets the requirements is available, it may then be configured for use to create the new system.

##### 3.5 Component adaptation and integration:
‚ñ™ If there is no off-the-shelf system, individual reusable components may be modified and new components developed. 
‚ñ™ These are then integrated to create the system.
